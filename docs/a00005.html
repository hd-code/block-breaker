Detailed Description

Defines the elements of a mesh.

A mesh is an array of vertices connected to triangles by a set of indices. Three indices form a single triangle. A vertex may contain more than only a position, e.g. normals, colors, or texture coordinates. Note that the position and these additional attributes are not defined in separate arrays but in one interleaved array. For example two vertices with position and normal are defined in this way: 

      X     Y     Z     NX    NY    NZ
0.    2.0f, 2.0f, 2.0f, 0.0f, 0.0f, 1.0f,
1.    4.0f, 2.0f, 4.0f, 0.0f, 1.0f, 0.0f,

The result is one single array: 

2.0f, 2.0f, 2.0f, 0.0f, 0.0f, 1.0f, 4.0f, 2.0f, 4.0f, 0.0f, 1.0f, 0.0f,

The combination of position and attributes is called the vertex layout. Note that each vertex is passed to a vertex shader. A vertex shader can be seen as a C-like procedure. The input of the procedure is a single vertex. Similar to a C/C++ procedure the vertex shader defines type and name of the arguments it expects. Those arguments have to match the vertex layout. If we call the combination of input arguments the input layout of the vertex shader then input layout and vertex layout have to match exactly. For example the two vertices from above have two elements, namely position and normal. Now take a look at a matching vertex shader declaration: 

struct VSInput
{
    float3 m_Position : POSITION;
    float3 m_Normal   : NORMAL;
};

PSInput VSShader(VSInput _Input)
{
    ...
}

As can be seen the vertex shader VSShader expects a struct VSInput as input. This struct has two members, which are 3D vectors with three float values each. The first vector is the position, the second vector is the normal. This matches the vertex layout. Do not be irritated by the fact that the struct _Input is only a single argument. It is just a more elegant notification for this: 

PSInput VSShader(float3 _Position : POSITION, float3 _Normal : NORMAL)
{
    ...
}

Each mesh is associated with a material. The material defines the look of the surface spawned over the mesh. A material is a combination of textures, vertex shader, pixel shader, and shader constants (global parameters for the shader programs). If you assign a material to the mesh, you have to take care that the input layout of the chosen vertex shader matches the vertex layout of the mesh.

Unfortunately this matching has to be ensured by the user and is not done automatically by the video driver or the GPU. For that reason the user has to define an array of gfx::SInputElement instances in the gfx::SMaterialInfo struct. Each instance defines a single argument of the input layout (and in this way also a single element of the vertex layout). For example to define the input layout of the vertex shader above you have to do the following: 

SMaterialInfo MaterialInfo;

MaterialInfo.m_NumberOfInputElements    = 2;
MaterialInfo.m_InputElements[0].m_pName = "POSITION"; 
MaterialInfo.m_InputElements[0].m_Type  = SInputElement::Float3;
MaterialInfo.m_InputElements[1].m_pName = "NORMAL";   
MaterialInfo.m_InputElements[1].m_Type  = SInputElement::Float3;

Note that the names POSITION and NORMAL are identical to the semantics in the VSInput struct. You can chose the name freely as long as the identifiers in the material info and in the vertex shader input are equal.
