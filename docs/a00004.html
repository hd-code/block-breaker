Detailed Description

Defines the look of a surface spawned over a mesh.

A material defines the look of a surface spawned over a mesh. Is is represented by a set of textures, a pixel shader, a vertex shader and a set of constant buffers. While each single vertex of a mesh is the local input of the vertex shader, the constant buffers provide global data which is accessible to all vertices and pixels. Note that you cannot mix constant buffers with vertex and pixel shader data. Instead you have to create separate buffers, even if this implies, that the same information (e.g. the light direction) has to be passed two times. The textures are usually sampled by the pixel shader. Sometimes it is useful to define a material with multiple textures, for example a color map, a normal map, and a height map. All meshes associated with the material have to fulfill a certain condition: The layout of each vertex has to match the input signature of the vertex shader. Otherwise the GPU will not be able to link the vertex shader to the vertices of the mesh. The user has to describe the input layout of the vertex shader not only by implementing the vertex shader itself but also by defining an array of gfx::SInputElement instances. The maximum number of input arguments of a vertex shader is 16.
