Detailed Description

Defines a single argument of the vertex shader input.

When drawing a mesh each vertex of the mesh is processed by a vertex shader. Shader programs are C-like procedures expecting input arguments and returning a result (e.g. a pixel shader returns the color of a pixel). A vertex shader is allowed to expect at most 16 arguments. The challenge for the graphics driver is to match an array of vertex data (mostly a sequence of float values) to the input arguments of the vertex shader. For example the driver has to know, how many sequent bytes of the array belong to one vertex. For that reason the user has to define one gfx::SInputElement instance for each input argument. 

MaterialInfo.m_NumberOfInputElements    = 2;
MaterialInfo.m_InputElements[0].m_pName = "POSITION"; 
MaterialInfo.m_InputElements[0].m_Type  = SInputElement::Float3;
MaterialInfo.m_InputElements[1].m_pName = "NORMAL";   
MaterialInfo.m_InputElements[1].m_Type  = SInputElement::Float3;

In this example two arguments are defined. The first argument represents the position of a vertex and contains 3 float values (i.e. 12 bytes). The second argument represents the normal of a vertex and contains also 3 float values. Consequently one vertex requires 24 bytes, whereas each 4 bytes represent one float value. The first three float values represent the position and the second three float values the normal of the vertex.

See gfx::SMeshInfo for further information.
